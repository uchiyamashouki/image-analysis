# コードレビュー（2026-04-13）

対象: `index.html`, `script.js`, `style.css`

## 主な問題点

1. **CDNバージョンが `@latest` 固定で再現性が低い**
   - `@mediapipe/tasks-vision@latest` を直接参照しており、将来の更新で突然挙動が変わるリスクがあります。

2. **入力パラメータの一部が実質未使用**
   - UIで入力できる「利き腕 (`throwingHand`)」は結果オブジェクトに保持されるだけで、解析ロジックに反映されていません。

3. **欠損補間が強制的で品質低下を隠す可能性**
   - 欠損値を前後補間で全面的に埋めるため、ランドマーク検出失敗が多い動画でも“連続した値”として扱われます。
   - 品質指標（検出率など）が無く、信頼性判断が難しいです。

4. **フレーム時刻生成で末尾重複が起こり得る**
   - `for (let t = 0; t <= duration; t += dt)` と `Math.min(t, duration)` の組合せで、duration近傍の時刻が重複する可能性があります。

5. **メトリクス計算で空配列を考慮していない箇所がある**
   - `Math.max(...xVals)` などが空配列時に `-Infinity` / `Infinity` になり、異常値が表示される可能性があります。

6. **可視化・CSVのエスケープ処理不足**
   - CSV組立で値を単純連結しており、将来的にカンマや改行を含む値が入った場合に壊れる可能性があります。

## 改善提案（優先順）

1. MediaPipe依存を固定バージョン化（`@latest` を廃止）
2. `throwingHand` を解析・表示へ反映（軸の符号統一など）
3. 検出品質メトリクス（検出成功率、補間率、低信頼フレーム数）追加
4. 時刻列生成を `Math.floor(duration * fps)` ベースに変更し重複排除
5. メトリクス前に `valid.length` チェックを追加し、N/A表示へフォールバック
6. CSVエスケープ関数を実装
